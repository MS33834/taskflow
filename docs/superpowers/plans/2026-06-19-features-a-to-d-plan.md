## A. 数据导入/导出

### 目标
用户能在设置页导出完整加密备份，并在新设备或重装后通过备份恢复全部数据。

### 方案讨论

**方案 A1：JSON 加密备份（推荐）**
- 导出：读取 SQLite 所有表 → 序列化为 JSON → 用主密钥 AES-GCM 加密 → 写入 `.taskflow-backup` 文件
- 导入：读取文件 → 解密 JSON → 清空当前数据库 → 插入数据 → 重新初始化
- 优点：人类可读结构、跨版本兼容性好、便于调试
- 缺点：备份文件体积比原生数据库大

**方案 A2：直接复制加密数据库文件**
- 导出：复制 `taskflow.db` 到用户选择的位置
- 导入：关闭当前数据库 → 替换文件 → 用相同主密钥重新打开
- 优点：最简单、100% 保留数据库状态
- 缺点：需要目标设备使用完全相同的 SQLCipher 版本和设置；跨版本可能不兼容；用户无法查看内容

**方案 A3：分卷加密备份**
- 大数据量时拆分成多个加密分卷
- 适用于未来数据量很大的场景
- 1.x 阶段可能过度设计

### 建议
1.x 阶段采用 **方案 A1（JSON 加密备份）**，兼顾可读性和安全性。

### 验收标准
- [ ] 设置页有「导出备份」按钮，弹出系统保存对话框
- [ ] 设置页有「导入备份」按钮，弹出系统选择对话框
- [ ] 备份文件用主密钥加密，无法被其他工具读取
- [ ] 导入后任务、保险库、设置全部恢复
- [ ] 导入前提示会覆盖现有数据并要求确认
- [ ] 提供自动备份开关（可选）

## B. 深色模式完整实现

### 目标
支持浅色/深色/跟随系统三种模式，并持久化用户选择。

### 方案讨论

**方案 B1：Tailwind dark 类（推荐）**
- 在 html 元素上切换 `dark` class
- Tailwind 配置 `darkMode: 'class'`
- 所有组件使用 `dark:bg-slate-900` 等前缀
- 优点：与当前技术栈一致、实现简单
- 缺点：需要逐个组件补全深色样式

**方案 B2：CSS 变量 + 主题对象**
- 定义两套 CSS 变量，通过切换 data-theme 属性换肤
- 优点：更灵活，未来可支持更多主题
- 缺点：需要重构现有颜色定义

### 建议
采用 **方案 B1**，快速补全现有组件的深色样式。

### 验收标准
- [ ] 设置页有「外观」选项：浅色、深色、跟随系统
- [ ] 选择后界面立即切换
- [ ] 重启应用后记住用户选择
- [ ] 所有页面（任务、保险库、设置、锁定界面）都支持深色
- [ ] 系统自动模式能响应 macOS/Windows 主题变化

## C. 截图保护

### 目标
防止敏感界面被截图、录屏或远程桌面工具抓取。

### 方案讨论

**方案 C1：Electron 窗口内容保护（推荐，先做这个）**
- `win.setContentProtection(true)`：阻止 Windows/macOS 上的屏幕捕获
- Linux 支持有限
- 优点：简单、跨平台（Windows/macOS）
- 缺点：Linux 不完全支持；不能防止物理相机拍摄屏幕

**方案 C2：界面模糊层**
- 在锁定或隐私模式下对内容区域做 CSS blur/pixelate
- 优点：所有平台可用、视觉效果直接
- 缺点：只是 UI 层保护，底层仍可能被截屏工具抓取

**方案 C3：结合 C1 + C2**
- 全局启用 `setContentProtection`
- 隐私模式下额外加视觉模糊
- 最完整，但工作量最大

### 建议
1.x 阶段采用 **方案 C1**，在锁定和保险库页面启用内容保护。后续可叠加 C2。

### 验收标准
- [ ] 打开保险库或锁定界面时，Windows/macOS 截图工具捕获为黑屏或提示受保护
- [ ] 提供设置项允许用户关闭截图保护（如果他们认为不需要）
- [ ] Linux 平台 gracefully degrade（不报错但可能无保护效果）

## D. 日历视图补全

### 目标
日历视图能显示任务分布、点击日期查看/添加任务。

### 方案讨论

**方案 D1：自定义日历组件（推荐）**
- 用 date-fns 生成月份网格
- 在对应日期格子显示任务点/数量
- 点击日期弹出任务列表或创建弹窗
- 优点：完全可控、无需额外依赖
- 缺点：需要自行实现交互和样式

**方案 D2：集成第三方日历库**
- 如 `react-big-calendar`、`fullcalendar`
- 优点：功能丰富
- 缺点：引入较大依赖、与现有隐私定位可能冲突

### 建议
采用 **方案 D1**，保持依赖精简和隐私可控。

### 验收标准
- [ ] 月视图显示当月所有日期
- [ ] 有任务的日期显示标记
- [ ] 点击日期弹出该日任务列表
- [ ] 支持切换上月/下月
- [ ] 支持回到今天
